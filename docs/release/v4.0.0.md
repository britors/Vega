# Vega 4.0.0

Vega 4.0.0 traz o `vega-cli`: uma interface de terminal completa, para
administrar um servidor via SSH sem depender de ambiente gráfico nenhum.

## vega-cli

Novo componente (`vega-cli/`, shell + `dialog`), paralelo ao `vega-gtk` e ao
`vegad` — reaproveita o daemon e o mesmo contrato D-Bus
(`org.lyraos.Vega1.*`) já usado pela interface gráfica, sem nenhuma mudança
de contrato. Cobre:

- Painel;
- Software (pacotes e repositórios);
- Backup e Pontos de Restauração;
- Hardware e Kernel;
- Usuários;
- Rede e Firewall (interfaces, proxy, VPN, firewall — Wi-Fi fica de fora,
  por não fazer sentido num servidor típico);
- Serviços;
- Data, Hora e Idioma;
- Armazenamento;
- Log do Sistema;
- Monitor do Sistema (CPU, memória, swap, disco, rede e processos, só em
  texto — sem gráficos).

Autorização de ações privilegiadas funciona via `pkttyagent`, registrado no
início da sessão do `vega-cli`: em vez de um agente gráfico (que sessões
SSH normalmente não têm), a senha é pedida em texto no próprio terminal.

Empacotado como um pacote novo, `vega-cli`, para Arch, Fedora, openSUSE e
Debian/Ubuntu. O instalador por curl ganhou a opção `VEGA_CLI_ONLY=1`, para
instalar só `vegad` + `vega-cli` num servidor headless, sem puxar
GTK4/libadwaita.

## Log do vegad em arquivo

`vegad` passa a exportar seu journal periodicamente para
`/var/log/vega/vegad.log` (com rotação semanal via logrotate), além do
journal (`journalctl -u vegad`), que continua funcionando normalmente e é
usado pelo próprio módulo Log do Sistema do `vega-cli`.

## Correções

- O catálogo do firewall usava `"printer"` como nome de serviço, mas o
  firewalld não reconhece esse ID — corrigido para `"ipp"`.
- `DateTime.Status` podia devolver `"(unset)"` como se fosse um layout de
  teclado válido, em sistemas sem layout X11 configurado — agora é tratado
  como "sem valor".
- `Kernel.ApplyBootConfig` falhava com "permission denied" ao escrever
  `/etc/default/grub` em sistemas com SELinux enforcing (ex.: openSUSE) —
  corrigido com um módulo de política SELinux dedicado, carregado só
  quando o sistema tem SELinux habilitado.
